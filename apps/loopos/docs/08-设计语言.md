# 回路OS · 设计语言

> **美学方向**：活体组织美学（Organic Living）
> **情感主张**：组织是活的生命体，治理是照护它的呼吸，不是监控它

---

## 一、美学主张

回路OS 的核心隐喻是**生命系统**：

- **回路 = 生物循环**（如血液循环、生态循环）
- **张力 = 生命信号**（如神经脉冲，组织通过张力感知自身）
- **阻塞点 = 需要滋养的部位**（像植物等待光照）
- **进化闭环 = 新陈代谢**（运转→张力→追踪→模式→宪章→回到运转）
- **组织宪章 = DNA**（决定组织如何生长，可随进化被改写）

这与回路制蓝图"组织即产品""张力即燃料""组织自身的呼吸"三条原则完全共振。

**为什么不是其他方向**：
- 制度美学太冷，让治理感觉像管控而非照护
- 工程美学（Linear/GitHub 风）虽专业但脱离"生命体"隐喻
- 编辑美学适合治理日志这类"长期记忆"，但日常运转页面太慢

活体组织美学的独特价值：让治理活动有温度。阻塞不是错误，是组织在呼吸。

## 二、色板（大地+有机绿+暖色）

### 主色板

| 角色 | 色值 | 用途 |
|------|------|------|
| **土壤 So深** | `#1F1B16` | 主文字、主背景（深色模式）/ 强调 |
| **泥土 So浅** | `#2A2520` | 深色模式次背景 |
| **暖米 So纸** | `#FAF6F0` | 浅色模式主背景（像纸张的温暖白）|
| **亚麻** | `#F0E9DC` | 卡片背景、分隔区 |
| **苔藓绿 主** | `#4A7C59` | 品牌色、主按钮、活跃状态 |
| **苔藓绿 浅** | `#7FA88E` | 次级强调、hover |
| **苔藓绿 极浅** | `#E8F0EA` | 标签背景、选中态 |

### 状态色（有机化处理，避免工业感的红绿）

| 状态 | 色值 | 命名隐喻 |
|------|------|---------|
| 待萌芽 OPEN | `#A8927C` 暖棕 | 种子，等待 |
| 生长中 IN_PROGRESS | `#4A7C59` 苔藓绿 | 正在生长 |
| 受阻 BLOCKED | `#C97B5E` 陶土红 | 需要光照 |
| 已成熟 RESOLVED | `#6B8E7F` 雾绿 | 完成，归于平静 |
| 紧急 L0.5 | `#B85450` 深朱红 | 需要立即照料 |

### 背景

- 浅色模式：`#FAF6F0` 暖米底，不用纯白（纯白太冷、太"医疗"）
- 深色模式：`#1F1B16` 土壤深底，不用纯黑
- 避免冷色调（冷蓝、冷灰），所有中性色都偏暖

## 三、字体

### 中文：思源宋体 / 思源黑体（Google Fonts: Noto Serif SC / Noto Sans SC）

- **思源宋体**：用于标题、关键数字、治理日志/宪章——有"沉淀感"，让长期记忆显得值得阅读
- **思源黑体**：用于正文、UI 控件、表单——清晰、现代

### 英文：Fraunces + Inter

- **Fraunces**（衬线，展示）：英文标题、状态标签、里程碑数字——有机的曲线和墨迹感
- **Inter**（无衬线）：英文正文、UI——可读性强

### 数字：Fraunces tabular

- 数字用 Fraunces 的等宽变体，对齐表格和看板的数字列

### 字号阶梯

```
display-xl  48px  思源宋体/Fraunces  主标题（回路地图）
display-l   36px  思源宋体/Fraunces  页面标题
h1          28px  思源宋体/Fraunces  分区标题
h2          22px  思源黑体/Inter      卡片标题
h3          18px  思源黑体/Inter      小标题
body        15px  思源黑体/Inter      正文
caption     13px  思源黑体/Inter      辅助文字
mono        13px  JetBrains Mono     编号(#047)、技术值
```

## 四、形态语言

### 圆角（有机，避免锋利直角）

| 元素 | 圆角 |
|------|------|
| 卡片 | 16px（柔和）|
| 按钮 | 10px |
| 输入框 | 8px |
| 标签/badge | full（胶囊）|
| 头像 | full（圆形）|

### 阴影（柔和、有深度，模拟光照）

不用锐利阴影。用**柔和大范围阴影**模拟环境光：

```css
--shadow-soft: 0 4px 20px -4px rgba(31, 27, 22, 0.08);
--shadow-card: 0 8px 30px -6px rgba(31, 27, 22, 0.10);
--shadow-hover: 0 12px 40px -8px rgba(31, 27, 22, 0.14);
```

### 边框

- 默认 1px `rgba(31,27,22,0.08)`（极淡的土壤色）
- 选中态：1.5px 苔藓绿
- 禁用：0.5px 虚线 暖棕

## 五、动效原则

### 核心隐喻：呼吸

组织在呼吸，界面也应呼吸。但克制——避免分散注意力的动画。

| 动效 | 实现 | 场景 |
|------|------|------|
| 页面加载 | 元素错峰淡入（staggered fade-in，80ms 间隔）| 首屏 |
| 卡片 hover | 轻微上浮（translateY -2px）+ 阴影加深 | 列表项 |
| 状态变化 | 颜色平滑过渡（200ms ease-out）| 阻塞点状态转移 |
| 新张力提交 | 从下方生长出现（scale 0.96→1 + opacity）| 新建记录 |
| 紧急阻塞 | 慢呼吸光晕（pulse，2s 周期）| L0.5 状态 |
| 回路地图 | 节点连线有轻微流动感（dashed offset animation）| 回路关系图 |

### 禁止

- ❌ 弹跳动画（bouncy/spring 过度）
- ❌ 视差滚动（分散注意力）
- ❌ 旋转加载图标（用骨架屏代替）
- ❌ 闪烁提醒（用呼吸光晕代替）

## 六、核心组件质感

### 卡片（阻塞点/张力/角色）

```
┌──────────────────────────────┐
│  ○ #047                  数据回路  │
│                                  │
│  预训练数据 D3 还未交付            │
│  ──────────────────────────      │
│  待萌芽 · 截止 7/9 · @翔宇         │
└──────────────────────────────┘
背景 #FAF6F0 · 边框 1px 暖灰 · 圆角 16px
hover 上浮 · 左侧 3px 苔藓绿条（活跃状态）
```

### 状态徽章（胶囊）

```
( 待萌芽 )  背景 #F0E9DC · 文字 #A8927C
( 生长中 )  背景 #E8F0EA · 文字 #4A7C59
( 受阻 )    背景 #FAEAE3 · 文字 #C97B5E
```

### 回路地图节点（React Flow）

圆形节点，大小代表回路规模，颜色代表健康度（绿=健康，陶土=预警）。连线用虚线，流动感（轻微的 dash offset 动画），像生命体内的能量流动。

### 空状态

不要冷冰冰的"暂无数据"。用温暖的引导：

```
        ◔
   还没有张力
   ─────────
   组织很平静，
   或者——
   有些该说的话还没说出口。
   → 提一个张力
```

## 七、排版原则

### 密度

- **默认中等密度**：信息清晰但不压迫
- **可切换紧凑/舒适**：列表页支持密度切换
- **治理会工作台用紧凑**：实时场景需要高信息密度

### 留白

慷慨留白，但不像 Stripe Press 那么夸张。治理工具需要在一屏看到更多信息，但留白要足以让眼睛呼吸。

## 八、暗色模式

- 背景 `#1F1B16` 土壤深（不用纯黑，纯黑太冷太"OLED"）
- 卡片 `#2A2520` 泥土浅
- 文字 `#FAF6F0` 暖米
- 状态色在暗色下略提亮，保证对比度
- 暗色模式是默认（长时间盯看，暗色更护眼）

## 九、与回路制语义的呼应

| 回路制概念 | 设计语言映射 |
|-----------|------------|
| 回路 = 生物循环 | 圆形节点、流动连线、苔藓绿 |
| 张力 = 生命信号 | 张力卡片有"脉冲"质感，提交时有生长动效 |
| 阻塞 = 需要滋养 | 陶土红色，"需要光照"而非"超时警告" |
| 进化闭环 = 新陈代谢 | 版本化用"生长年轮"视觉（治理日志/宪章）|
| 组织宪章 = DNA | 宪章页用衬线字体，像古老的有机法则 |
| 归属 = 根系 | 人员卡片的归属用"扎根于 XX 回路"表达 |

---

## 十、不要做的事

- ❌ 不用纯白背景（太冷）
- ❌ 不用纯黑背景（太冷）
- ❌ 不用冷蓝/冷灰中性色
- ❌ 不用 Material 风格的锐利阴影
- ❌ 不用 emoji（用有机 SVG 图标）
- ❌ 不用 Linear 风格的"工程感"（那是另一个美学方向）
- ❌ 不让动画分散注意力（呼吸可以，闪烁不行）
