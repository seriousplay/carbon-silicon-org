# LoopOS Goals

Last updated: 2026-07-14

## Coordination Rules

- This file is the shared route map for the long-running LoopOS project.
- The main thread owns goals, constraints, decisions, project state, and evidence gates.
- Implementation work should be delegated to subagents or separate threads.
- Worker responses must report only: conclusion, changes, evidence, blockers.
- Use a separate thread for work that may need future review or audit.
- Use a subagent for temporary investigation or implementation tasks.
- Browser, desktop app, permission, credential, or device-state tests must run in a local thread.
- After each milestone, open a separate audit thread to compare this file with the actual state, run `/review` on implementation, update this route map, then activate the next goal.
- Maintain `progress-dashboard.html` when multiple milestones or multiple workers are active.
- Do not declare a milestone complete until the agreed evidence exists.
