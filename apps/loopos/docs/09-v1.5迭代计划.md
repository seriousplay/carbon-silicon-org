# 回路OS · V1.5 迭代计划（Sprint 1+2 合并）

> **状态**：执行中
> **日期**：2026-07-08
> **目标**：6.27/10 → 7.8+/10，从"静态记录工具"进化为"会主动说话的活体组织"
> **依据**：review/v1/ 三视角交叉验证（工程/产品/迭代策略）

---

## 一、V1 共识级问题（本迭代要解决的）

| # | 问题 | 三视角证据 | 本迭代解决方式 |
|---|------|-----------|--------------|
| 🔴1 | AI 集成零进展 | 架构4.0/产品1.0/战略零 | 可切换多 provider 抽象层 + 张力翻译 + 会议议程 + 守护报告 |
| 🔴2 | 通知系统缺失 | 模型存在无写入代码 | 通知创建 + 站内 inbox + 侧边栏徽章 + 邮件 |
| 🔴3 | 定时任务零实现 | scanEscalations 空函数 | 独立 worker（node-cron）+ 完整升级扫描 |
| 🔴4 | 权限校验缺失 | transitionBlocker 不验证 owner | 所有写操作补权限校验 |
| 🟠5 | 编辑功能仅20% | 创建后不可改 | 阻塞点/回路/张力/角色编辑 |
| 🟠6 | 无搜索/筛选 | 超30条人眼扫描 | 全局搜索 ⌘K + 列表筛选 |
| 🟠7 | 移动端不可用 | 侧边栏消失 | 移动端 Sheet 导航 |
| 🟠8 | 回路间接口无UI | Schema完整无页面 | 接口契约管理 UI |

## 二、关键架构决策（用户确认）

1. **AI provider**：Vercel AI SDK 多 provider 抽象层（可切换 OpenAI/Anthropic/阶跃），环境变量配置
2. **定时任务**：独立 Node worker 进程（node-cron），不依赖 Vercel
3. **范围**：Sprint 1+2 合并，约 50h，目标 7.8/10

## 三、执行顺序（依赖关系驱动）

```
P0 基础设施（先做，其他依赖它）
  ├── AI provider 抽象层（lib/ai/）
  ├── 通知子系统（lib/notifications/）
  └── 权限 helper（lib/permissions.ts）

P0 功能（依赖基础设施）
  ├── AI 张力翻译（依赖 AI 抽象层）
  ├── 通知站内信 + 徽章（依赖通知子系统）
  ├── 独立 worker + scanEscalations（依赖通知子系统）
  ├── 权限校验补全（依赖权限 helper）
  ├── 状态转移刷新 + 事务安全
  └── 编辑功能

Sprint 2 功能
  ├── 会议议程 AI 生成（依赖 AI 抽象层）
  ├── 会后守护者报告（依赖 AI 抽象层）
  ├── 全局搜索 ⌘K
  ├── 移动端导航
  ├── 回路间接口 UI
  └── 邮件通知（Resend）
```
