## 一、核心概念：回路 vs 项目

```
回路 Circle  = 持续存在的组织单元（器官）
项目 Project = 有始有终的工作（生理过程）
```

- 回路是**结构层**（治理会修改），包含角色和子回路
- 项目是**工作层**（战术会分配），包含行动项
- 项目归属在一个回路下，完成即结束
- 一个回路可同时进行多个项目

## 二、战术会议：项目 + 行动项跟踪

### 数据模型

```
Project（新增）
  ├── name: 项目名称
  ├── goal: 项目目标
  ├── status: ACTIVE / COMPLETED / PAUSED
  ├── circleId: 归属回路
  ├── sourceTensionId: 发起此项目的张力
  └── tensions[]: 项目下的行动项（已有 Tension 加 projectId 字段）

Tension（加字段）
  └── projectId?: 归属的项目（可选）
```

### 战术会流程

```
张力提出 → 讨论 → 决定处理方式：
  ├─→ 分配为独立行动项（当前已有的流程）
  └─→ 发起项目（新建 Project）→ 项目下拆解多个行动项
```

### 跟踪

- 项目页：展示项目目标 + 所有子行动项的进度（完成率）
- 看板：可按项目筛选行动项
- 概览：进行中项目数

## 三、治理会议：简化三步法

### 流程

```
1. 张力提出
     ↓ 提案人将张力转化为具体的治理提案
2. 提案（Proposal）
     ├─ 提案类型：新增角色 / 修改角色 / 废弃角色 / 新增回路 / 修改回路 / 归属变更
     ├─ 提案内容：具体要改什么（结构化）
     └─ 提案理由：为什么
     ↓ 其他参会人有时间反应
3. 采纳或反对
     ├─ 无反对 → 采纳 → ★自动执行结构修改
     └─ 有反对 → 标记反对意见 → 整合后重新提案或搁置
```

### 数据模型

```
GovernanceProposal（新增）
  ├── tensionId: 来源张力
  ├── meetingId: 在哪次治理会讨论
  ├── type: 角色新增/修改/废弃/回路新增/修改/归属变更
  ├── targetId: 修改的对象（角色ID/回路ID/人员ID）
  ├── proposedChange: 具体修改内容（JSON：字段→新值）
  ├── rationale: 提案理由
  ├── status: PROPOSED / ADOPTED / OBJECTED / WITHDRAWN
  ├── objections[]: 反对意见
  └── adoptedAt: 采纳时间
```

## 四、治理决策自动落地到结构

### 核心原则

**决策不只是文字记录，采纳后必须自动修改回路/角色定义。**

### 落地规则

| 提案类型 | 采纳时执行 | 审计记录 |
|---------|-----------|---------|
| 角色新增 | `RoleDef.create(proposedChange)` | ChangeLog(ROLE_CREATED) |
| 角色修改 | `RoleDef.update(targetId, proposedChange)` | ChangeLog(ROLE_MODIFIED) |
| 角色废弃 | `RoleDef.update(targetId, {status: ARCHIVED})` | ChangeLog(ROLE_ARCHIVED) |
| 回路新增 | `Circle.create(proposedChange)` | ChangeLog(CIRCLE_CREATED) |
| 回路修改 | `Circle.update(targetId, proposedChange)` | ChangeLog(CIRCLE_MODIFIED) |
| 归属变更 | `Person.update(targetId, {homeCircleId})` | ChangeLog(HOME_CHANGE) |

每条提案采纳时，在事务中同时：
1. 执行结构修改
2. 创建 DecisionRecord（决策记录）
3. 创建 ChangeLog（变更审计）
4. 更新张力状态

### 完整闭环

```
治理会
  ├── 张力 → 提案 → 采纳
  │     ↓ 自动执行
  │   角色/回路结构修改
  │     ↓ 自动记录
  │   DecisionRecord + ChangeLog
  │     ↓ 张力状态更新
  │   tension.status = RESOLVED
  │
  └── 回路地图/角色详情/人员页 自动反映新结构
```
