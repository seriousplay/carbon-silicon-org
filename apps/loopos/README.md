# 回路治理组织操作系统（回路OS）

> **定位**：为某 AI 团队定制的内部治理工具，把"回路制"方法论真正跑起来。自建网页 SaaS，不依赖飞书。
> 远期愿景是沉淀为可复用的组织治理操作系统，但当前阶段专注把一个团队治理好。

> ⚠ **项目状态**：v4，架构转向自建网页（Next.js + PostgreSQL）。详见 `docs/07-技术架构与栈选型.md`。

## 为什么

很多快速扩张的 AI 团队都会撞上同一个矛盾：「权力-责任-资源」三角失衡，外化为部门墙 → 协作失效 → 产品掉队。"回路制"给出的解药是把部门墙拆掉、按真实生产链路重编组为 4+1 个回路，并配以双层会议、追踪表、组织宪章等治理机制。

但这些机制全部停留在**文档描述层**，没有落地到任何工具载体。飞书在文档里只被当作"群"和"文档"用过，多维表格的治理能力完全没被激活。

**回路OS 要填补的，正是这个空白**——让回路制从"咨询方案的文字"变成"组织每天在飞书里呼吸的系统"。

## 核心理念（四条不变量）

1. **回路制为主，Holacracy 为辅** —— 混合模型，非 Holacracy 实现。继承回路制的归属模型和生产链路编组，吸收 Holacracy 的角色解耦和张力驱动治理。当两者冲突时，回路制优先。
2. **状态在表格，交互在对话** —— 人和"@回路OS"对话，结构化结果沉淀回多维表格。
3. **AI 做教练，不做决策者** —— Agent 承担外部教练三角色（会议守护者/张力翻译者/机制建设者），绝不替回路负责人或决策者拍板。**AI 能力有明确边界和降级策略**（见 `docs/06-AI能力边界与降级.md`）。
4. **为某团队深度定制，远期可复用** —— 当前是内部工具，不追求通用产品化。"可复用产品"是方法论被验证后的远期愿景。

## 系统架构（四层）

```
┌─────────────────────────────────────────────────────────────┐
│  L4 进化层   组织宪章(季度/Wiki版本化) ← 治理日志(月度/AI草稿)  │
├─────────────────────────────────────────────────────────────┤
│  L3 节奏层   战术会(30min) │ 治理会(90min) │ 战略回路(双周)     │
├─────────────────────────────────────────────────────────────┤
│  L2 感知层   张力/阻塞点 │ 冲突升级L1-L4 │ 教练Agent │ 防双线守护 │
├─────────────────────────────────────────────────────────────┤
│  L1 状态层   7张主表：回路/角色/人员/张力-阻塞/追踪/接口/指标    │
└─────────────────────────────────────────────────────────────┘
   飞书多维表格(Base)=状态  机器人@回路OS=交互  文档/Wiki=记忆  日历/会议=节奏
```

## 关键设计决策

### 归属 vs 角色（数据模型的灵魂）

这是回路制与 Holacracy 融合时最容易混淆的点，我们已澄清：

- **归属（Accountability）** = 绩效和日常优先级由谁决定 → **唯一**（回路制 v2.0 P0-1 的硬约束，消灭双线汇报）
- **角色（Role）** = 承担什么职责 → **一人多角色**（Holacracy 把人和角色解耦的精髓）

所以：
- 归属确认卡 **1 人 1 卡**，只确认归属的唯一性
- 1 个人可承担 **N 个角色**，由角色表承载
- 跨回路支援角色必须是**契约化**的（经"回路间接口表"记录），派活权默认仍归主回路负责人
- 系统内置**三道防双线汇报守护**，否则消灭双线汇报的硬约束会失效

详见 `docs/00-系统设计总览.md`。

## 交付路线（三批）

> ⚠ review 修正：v2 称"每批可独立运转"不准确。批次1 实际是"建表+基础查询可用"，不形成治理闭环。批次间增加用户行为里程碑。

| 批次 | 范围 | 完成标志（诚实描述） |
|------|------|----------|
| **批次1：骨架** | L1 状态层 + L2 基础 | 建表完成 + 基础查询可用（看见回路结构、提张力、查归属）。**不形成治理闭环。** |
| **批次2：节奏** | L3 + L2 升级 | 回路真正转起来，闭环率可追踪。前置：日均提张力≥3条 |
| **批次3：进化** | L4 + Agent 完整 | 组织持续进化。前置：闭环率≥50%、AI 张力翻译准确率≥85% |
